교안 핵심 내용 요약

1. 연결 리스트의 핵심 구조 (Slide 12-14)

연결 리스트는 데이터 필드와 다음 노드를 가리키는 포인터 필드로 구성된 노드(Node)들의 집합이다. 메모리의 연속된 공간을 점유하는 배열과 달리, 연결 리스트는 동적 메모리 할당을 통해 런타임에 필요한 만큼 노드를 생성하고 연결하므로 메모리 효율이 극대화된다.

2. 삽입 및 삭제 연산 (Slide 15-17)

원소 삽입/삭제 시 배열은 O(N)의 이동 복잡도가 발생하나, 연결 리스트는 포인터 연결 변경만으로 O(1) 시간 내에 삽입/삭제가 수행된다. 다만, 특정 위치를 탐색하는 데에는 처음부터 노드를 추적해야 하므로 O(N)의 탐색 복잡도가 요구된다.

AI 피드백 총평

우수한 부분: 배열의 연속적인 메모리 할당 및 삽입/삭제 시 인덱스 조정을 위한 데이터 이동(O(N))을 언급하고, 연결 리스트가 링크 조정을 통해 삽입/삭제가 신속함(O(1))을 명확하게 잘 기술하였습니다.

아쉬운 부분 / 감점 사유: 연결 리스트가 동적 메모리 할당(malloc/free)을 기반으로 작동한다는 핵심 동작 원리를 기재하지 않았습니다. 또한 다음 데이터를 가리키기 위해 노드(Node) 구조 속의 포인터(Pointer) 필드를 활용한다는 개념 언급이 누락되었습니다.

연결 리스트의 구조

  • 노드 (Node) 중심의 메모리 할당 방식
    • 데이터 필드 (Data Field): 실제 저장하고자 하는 데이터 값
    • 포인터 필드 (Pointer Field): 다음 노드의 주소를 가리키는 링크 필드
  • 동적 메모리 할당 (Dynamic Memory Allocation)
    • 배열과 달리 실행 시간(Runtime)에 필요한 만큼의 메모리를 malloc() 등으로 할당
    • 불연속적인 메모리 공간 분산 배치 가능
